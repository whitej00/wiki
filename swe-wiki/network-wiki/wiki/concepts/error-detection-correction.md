# 오류 검출과 정정 (Error Detection and Correction)

링크 계층에서 프레임의 **비트 오류를 검출하고 정정**하는 기법. 신호 감쇠와 전자기 노이즈로 인해 비트가 뒤집힐 수 있으며, 이를 감지하기 위해 송신 노드가 **오류 검출 비트(EDC)**를 데이터에 추가한다.

> 어떤 오류 검출 기법도 100% 완벽하지 않다. 검출되지 않는 비트 오류(**undetected bit errors**)가 존재할 수 있다.

## 1. 패리티 검사 (Parity Checks)

### 1차원 패리티 (Single-Bit Parity)

- 데이터 d비트에 1개의 패리티 비트를 추가하여 d+1 비트에서 1의 개수가 짝수(even parity) 또는 홀수(odd parity)가 되도록 함
- **단일 비트 오류 검출** 가능
- 짝수 개의 비트 오류는 검출 불가
- 버스트 오류 환경에서 미검출 확률 ~50%

### 2차원 패리티 (Two-Dimensional Parity)

- d비트를 i행 × j열 행렬로 배치하고, 각 행과 열에 패리티 비트 추가
- 총 i + j + 1 개의 패리티 비트 사용
- **단일 비트 오류를 검출하고 정정** 가능 (행·열 교차점으로 위치 식별)
- 두 비트 오류는 **검출**은 되지만 **정정은 불가**

> 수신측에서 오류를 검출하고 정정까지 하는 능력을 **전방 오류 정정(FEC, Forward Error Correction)**이라 한다. FEC는 재전송 지연을 피할 수 있어, 실시간 애플리케이션이나 긴 전파 지연 링크(위성 등)에서 유용하다.

## 2. 체크섬 (Checksumming Methods)

- 데이터를 k비트 정수의 시퀀스로 취급하여 합산
- **인터넷 체크섬(Internet Checksum)**: 16비트 정수 합의 1의 보수 (TCP, UDP, IP에서 사용)
- 장점: 계산이 간단하여 **소프트웨어 구현에 적합** (전송 계층)
- 단점: CRC에 비해 오류 검출 능력이 약함 (16비트만 사용)

## 3. 순환 중복 검사 (CRC, Cyclic Redundancy Check)

**다항식 코드(Polynomial Code)**라고도 불리며, 현대 링크 계층에서 가장 널리 사용되는 오류 검출 기법이다.

### 동작 원리

1. 송수신자가 r+1 비트의 **생성자(Generator) G**에 합의 (최상위 비트 = 1)
2. 송신: 데이터 D에 r비트의 CRC 비트 R을 추가하여 d+r 비트 전송
3. R은 `D · 2^r XOR R`이 G로 나누어떨어지도록 선택
4. 즉, `R = remainder(D · 2^r / G)` (modulo-2 나눗셈)
5. 수신: d+r 비트를 G로 나누어 **나머지가 0이면 정상**, 0이 아니면 오류 검출

### CRC의 오류 검출 능력

- r+1 비트 미만의 **모든 버스트 오류** 검출
- r 비트 이하의 **모든 연속 비트 오류** 검출
- r+1 비트 이상의 버스트 오류도 1 − 0.5^r 확률로 검출
- **모든 홀수 개 비트 오류** 검출 가능

### CRC 표준

- **CRC-32**: 32비트 생성자, IEEE 802 링크 계층 프로토콜(이더넷 등)에서 사용
  - G_CRC-32 = `100000100110000010001110110110111`

### 왜 계층마다 다른 오류 검출 기법을 쓰는가

| 계층 | 기법 | 이유 |
|------|------|------|
| 전송 계층 (TCP/UDP) | 체크섬 | 소프트웨어 구현 → 빠르고 간단해야 함 |
| 링크 계층 (이더넷) | CRC | 하드웨어(NIC) 구현 → 복잡한 연산도 빠르게 처리 가능 |
