# 라우팅 알고리즘 (Routing Algorithms)

네트워크의 라우터들을 통해 송신자에서 수신자까지의 **최소 비용 경로(least-cost path)**를 결정하는 알고리즘. 컨트롤 플레인의 핵심 구성 요소이며, [OSPF](../protocols/ospf.md)와 [BGP](../protocols/bgp.md) 같은 라우팅 프로토콜의 기반이 된다.

## 그래프 모델

네트워크를 그래프 G = (N, E)로 추상화한다:
- **N**: 노드 집합 (라우터)
- **E**: 에지 집합 (링크)
- **c(x, y)**: 에지 (x, y)의 비용. 에지가 없으면 c(x, y) = ∞

비용은 물리적 거리, 링크 속도, 금전적 비용 등을 반영할 수 있다. 무방향 그래프를 가정하면 c(x, y) = c(y, x).

**경로(path)**: 노드 시퀀스 (x₁, x₂, ..., xₚ)에서 인접 쌍이 모두 E에 속함. 경로 비용은 구성 에지 비용의 합.

## 분류

### 1. 중앙집중형 vs 분산형

| | 중앙집중형 (Centralized) | 분산형 (Decentralized) |
|---|---|---|
| **정보** | 전체 토폴로지 + 모든 링크 비용 | 직접 연결된 이웃 정보만 |
| **계산** | 한 곳에서 수행 | 각 노드에서 반복적으로 수행 |
| **대표** | Link-State (LS) | Distance-Vector (DV) |

### 2. 정적 vs 동적

- **정적(Static)**: 관리자가 수동으로 경로 변경. 드물게 갱신.
- **동적(Dynamic)**: 네트워크 트래픽이나 토폴로지 변화에 반응하여 자동 갱신. 적응적이지만 라우팅 루프·진동에 취약.

### 3. 부하 감응 vs 부하 무관

- **부하 감응(Load-sensitive)**: 링크 혼잡도를 비용에 반영. 초기 ARPAnet에서 사용했으나 문제 발생.
- **부하 무관(Load-insensitive)**: 링크 비용이 현재 트래픽과 무관. **현재 인터넷(OSPF, BGP)은 모두 부하 무관**.

## Link-State (LS) 알고리즘 — Dijkstra

### 전제 조건
모든 노드가 **link-state broadcast**를 통해 전체 네트워크 토폴로지를 파악한다. OSPF가 이 방식을 사용한다.

### 알고리즘 (소스 노드 u)

**표기법**:
- D(v): 소스 u에서 v까지 현재 알려진 최소 비용
- p(v): u→v 최소 비용 경로에서 v의 직전 노드
- N': 최소 비용이 확정된 노드 집합

```
초기화:
  N' = {u}
  모든 노드 v: D(v) = c(u,v) (이웃이면), 아니면 ∞

Loop:
  N'에 없는 노드 중 D(w)가 최소인 w 선택
  w를 N'에 추가
  w의 이웃 v (v ∉ N')마다:
    D(v) = min(D(v), D(w) + c(w,v))
until N' = N
```

### 복잡도
- 기본 구현: **O(n²)** (n = 노드 수)
- 힙 사용 시: **O(n log n)**

### 진동 문제 (Oscillation)
혼잡 기반(부하 감응) 링크 비용 사용 시, 모든 라우터가 동시에 LS를 실행하면 트래픽이 한 방향으로 쏠렸다가 반대로 쏠리는 **진동**이 반복된다.

해결책:
- 링크 비용을 부하와 무관하게 설정 (현재 인터넷 방식)
- LS 실행 시점을 라우터마다 랜덤화 (자기동기화 방지)

## Distance-Vector (DV) 알고리즘 — Bellman-Ford

### Bellman-Ford 방정식

$$d_x(y) = \min_v \{ c(x, v) + d_v(y) \}$$

x에서 y로의 최소 비용은, x의 모든 이웃 v에 대해 "x→v 링크 비용 + v→y 최소 비용"의 최솟값이다.

### 특성
- **분산(distributed)**: 이웃으로부터만 정보 수신
- **반복적(iterative)**: 변화가 없을 때까지 계속
- **비동기(asynchronous)**: 노드가 동시에 동작할 필요 없음
- **자기종료(self-terminating)**: 변화 없으면 자연히 멈춤

### 알고리즘 (노드 x)

```
초기화:
  모든 목적지 y: D_x(y) = c(x,y) (이웃이면), ∞ (아니면)
  이웃 w에게 거리벡터 D_x 전송

Loop (forever):
  wait (링크 비용 변경 or 이웃 거리벡터 수신)
  모든 y: D_x(y) = min_v { c(x,v) + D_v(y) }
  if D_x 변경됨:
    모든 이웃에게 D_x 전송
```

### Count-to-Infinity 문제
링크 비용이 **증가**할 때, 두 노드가 서로의 (이미 무효한) 거리벡터를 참조하며 비용을 1씩 증가시키는 루프가 발생한다. 수렴까지 매우 오래 걸릴 수 있다 (예: 비용 4→60 변경 시 44회 반복).

### Poisoned Reverse
z가 y를 통해 x에 도달하는 경우, z는 y에게 D_z(x) = ∞라고 거짓 보고한다. y는 z를 통한 경로가 없다고 판단하여 2노드 루프를 방지한다.

**한계**: 3개 이상 노드가 관여하는 루프는 감지하지 못한다.

## LS vs DV 비교

| 기준 | Link-State (LS) | Distance-Vector (DV) |
|---|---|---|
| **메시지 복잡도** | O(\|N\| · \|E\|) 메시지 필요 | 이웃 간만 교환, 수렴 시간 가변 |
| **수렴 속도** | O(n²), 비교적 빠름 | 느림, 라우팅 루프 가능 |
| **강건성** | 각 노드 독립 계산 (오류 격리) | 오류가 네트워크 전체로 전파 가능 |
| **대표 프로토콜** | [OSPF](../protocols/ospf.md), IS-IS | RIP, BGP (DV 계열) |

어느 알고리즘도 절대적 우위는 없으며, 인터넷에서 **둘 다 사용**된다.
