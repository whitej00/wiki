# 프로토콜 계층 (Protocol Layers)

네트워크 프로토콜과 이를 구현하는 하드웨어/소프트웨어를 **계층(layer)**으로 조직화한 아키텍처. 각 계층은 아래 계층의 서비스를 사용하여 위 계층에 서비스를 제공한다.

## 왜 필요한가

인터넷은 수많은 프로토콜, 종단 시스템, 패킷 스위치, 링크 매체로 구성된 극도로 복잡한 시스템이다. 계층화는 이 복잡성을 관리 가능한 모듈로 분할하고, 각 계층의 구현을 독립적으로 변경할 수 있게 한다.

## 인터넷 프로토콜 스택 (5계층)

| 계층 | PDU 명칭 | 역할 | 주요 프로토콜 |
|---|---|---|---|
| **Application** | 메시지(Message) | 네트워크 애플리케이션과 프로토콜 | HTTP, SMTP, FTP, DNS |
| **Transport** | 세그먼트(Segment) | 프로세스 간 데이터 전달 | [[TCP]], [[UDP]] |
| **Network** | 데이터그램(Datagram) | 호스트 간 데이터그램 라우팅 | [[IP]], 라우팅 프로토콜 |
| **Link** | 프레임(Frame) | 인접 노드 간 데이터 전달 | 이더넷, WiFi, DOCSIS |
| **Physical** | 비트(Bit) | 링크를 통한 개별 비트 이동 | 매체 종속적 |

## OSI 7계층 모델과의 비교

OSI 모델은 인터넷 스택의 Application 계층을 3개로 세분화:
- **Session** — 세션 동기화, 체크포인팅
- **Presentation** — 데이터 압축, 암호화, 표현 변환

인터넷 아키텍처에서는 이러한 기능이 필요하면 애플리케이션 개발자가 직접 구현한다.

## 계층별 구현 위치

- **호스트(종단 시스템)**: 5개 계층 모두 구현
- **라우터**: Network, Link, Physical (3개 계층)
- **링크 계층 스위치**: Link, Physical (2개 계층)

## 장단점

**장점:**
- 모듈성: 각 계층의 구현을 독립적으로 변경 가능
- 구조화된 논의와 설계 가능

**단점:**
- 계층 간 기능 중복 가능 (예: 오류 복구를 링크 계층과 전송 계층 모두에서 수행)
- 한 계층이 다른 계층의 정보를 필요로 할 수 있음 (계층 분리 위반)
