# 다중화와 역다중화 (Multiplexing & Demultiplexing)

전송 계층이 네트워크 계층의 **호스트 간(host-to-host)** 전달 서비스를 **프로세스 간(process-to-process)** 전달 서비스로 확장하는 핵심 기능이다. 모든 컴퓨터 네트워크에서 필요하다.

## 정의

- **역다중화(Demultiplexing)**: 수신 호스트의 전송 계층이 네트워크 계층으로부터 세그먼트를 받아, 세그먼트 헤더의 필드를 검사하여 **올바른 소켓**으로 전달하는 작업
- **다중화(Multiplexing)**: 송신 호스트의 전송 계층이 여러 소켓에서 데이터를 모아, 헤더 정보를 붙여 세그먼트를 생성하고 네트워크 계층으로 전달하는 작업

## 왜 필요한지

하나의 호스트에서 여러 네트워크 애플리케이션 프로세스가 동시에 실행될 수 있다. 전송 계층은 네트워크 계층에서 올라온 데이터를 올바른 프로세스에 전달해야 한다. 프로세스는 **소켓(Socket)**을 통해 네트워크와 데이터를 주고받으며, 전송 계층은 실제로 프로세스가 아닌 **소켓**에 데이터를 전달한다.

## 포트 번호

다중화/역다중화를 위해 각 세그먼트에는 **출발지 포트 번호(Source Port Number)**와 **목적지 포트 번호(Destination Port Number)** 필드가 있다.

- 포트 번호: **16비트** 정수 (0~65535)
- **Well-known 포트** (0~1023): 예약된 포트. HTTP(80), FTP(21), DNS(53), SMTP(25) 등. RFC 1700, IANA 관리
- 1024~65535: 애플리케이션이 자유롭게 사용

출발지 포트 번호는 응답 시 **"회신 주소"** 역할을 한다.

## 비연결형 다중화/역다중화 (UDP)

[UDP](../protocols/udp.md) 소켓은 **(목적지 IP 주소, 목적지 포트 번호)** 2-tuple로 식별된다.

따라서:
- 출발지 IP나 출발지 포트가 달라도, **같은 목적지 IP·포트**를 가진 세그먼트는 같은 소켓으로 전달
- 출발지 포트는 응답 전송 시 목적지 포트로 사용됨

UDP 소켓 생성 시 전송 계층이 자동으로 1024~65535 범위의 포트 번호를 할당하거나, `bind()`로 특정 포트 지정 가능.

## 연결 지향 다중화/역다중화 (TCP)

[TCP](../protocols/tcp.md) 소켓은 **(출발지 IP 주소, 출발지 포트 번호, 목적지 IP 주소, 목적지 포트 번호)** 4-tuple로 식별된다.

따라서:
- 같은 목적지 포트(예: 80)에 도착하더라도, 출발지 IP나 출발지 포트가 다르면 **서로 다른 소켓**으로 전달
- 웹 서버는 클라이언트마다 별도의 연결 소켓을 생성

### 웹 서버와 TCP

- 서버는 포트 12000 등에서 연결 대기하는 **환영 소켓(Welcoming Socket)** 보유
- 클라이언트 연결 요청(SYN) 수신 시, 4-tuple로 식별되는 **새 연결 소켓** 생성
- 고성능 웹 서버는 프로세스 하나에 스레드 여럿을 사용, 각 연결 소켓을 별도 스레드에 할당
- 비지속(non-persistent) HTTP: 요청/응답마다 새 소켓 생성·폐기 → 성능 부담

## 포트 스캐닝

서버의 어떤 포트가 열려 있는지 조사하는 기법. **nmap**이 대표적 도구:
- TCP 포트: SYN 세그먼트 전송 → SYNACK(열림), RST(닫힘), 무응답(필터링)
- UDP 포트: UDP 세그먼트 전송 → 응답 UDP(열림), ICMP unreachable(닫힘), 무응답(열림 또는 필터링)

시스템 관리자에게는 유용한 보안 감사 도구이지만, 공격자가 취약점 탐색에 악용할 수 있다.
