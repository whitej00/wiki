# 패킷 교환 (Packet Switching)

데이터를 **패킷(packet)**이라는 작은 단위로 분할하여 네트워크를 통해 독립적으로 전송하는 방식. 현대 인터넷의 핵심 데이터 전달 메커니즘이다.

## 왜 필요한가

회선 교환(Circuit Switching)은 통신 시간 동안 자원을 전용 예약하므로, 버스트(bursty) 트래픽이 많은 데이터 통신에서는 자원 낭비가 심하다. 패킷 교환은 자원을 **온디맨드(on demand)**로 공유하여 훨씬 더 많은 사용자를 지원할 수 있다.

## 핵심 메커니즘

### 저장 후 전달 (Store-and-Forward)
패킷 스위치(라우터)는 패킷의 **모든 비트를 수신한 후**에야 다음 링크로 전송을 시작한다. N개의 링크(전송률 R)를 거쳐 L비트 패킷을 전송할 때:

```
d_end-to-end = N × (L/R)
```

### 큐잉 지연과 패킷 손실
- 각 패킷 스위치에는 **출력 버퍼(output buffer/queue)**가 존재
- 링크가 사용 중이면 도착 패킷은 큐에서 대기 → **큐잉 지연(queuing delay)**
- 버퍼가 가득 차면 패킷이 **드롭(drop)** → **패킷 손실(packet loss)**

### 포워딩 테이블과 라우팅
- 모든 종단 시스템은 **IP 주소**를 가짐
- 라우터는 패킷의 목적지 IP 주소의 일부를 **포워딩 테이블(forwarding table)**에서 조회하여 출력 링크를 결정
- **라우팅 프로토콜**이 포워딩 테이블을 자동으로 설정

## 패킷 교환 vs [회선 교환](circuit-switching.md)

| 비교 항목 | 패킷 교환 | 회선 교환 |
|---|---|---|
| 자원 할당 | 온디맨드 (통계적 다중화) | 사전 예약 (전용) |
| 대역폭 공유 | 효율적 | 비효율적 (유휴 시간 낭비) |
| 지연 특성 | 가변적 (큐잉 지연) | 일정 (보장된 전송률) |
| 사용자 수 | 더 많이 지원 가능 | 제한적 |
| 설정 시간 | 없음 | 회선 설정 필요 |
| 적합한 용도 | 데이터 통신 (버스트 트래픽) | 실시간 음성 (일정 전송률) |

> **예시**: 1 Mbps 링크, 사용자당 100 kbps (활성률 10%). 회선 교환은 10명만 지원. 패킷 교환은 35명이 동시에 11명 이상 활성일 확률이 0.0004로, 사실상 **3배 이상** 사용자를 지원한다.
