# 암호화 (Cryptography)

네트워크 보안의 초석. 데이터를 변환하여 침입자가 가로채더라도 내용을 알 수 없게 만드는 기술이다. 기밀성뿐 아니라 인증, 메시지 무결성, 부인 방지에도 핵심적으로 사용된다.

## 기본 용어

| 용어 | 설명 |
|------|------|
| **평문(Plaintext/Cleartext)** | 원본 메시지 |
| **암호문(Ciphertext)** | 암호화된 메시지 |
| **암호화 알고리즘(Encryption Algorithm)** | 평문 + 키 → 암호문 |
| **복호화 알고리즘(Decryption Algorithm)** | 암호문 + 키 → 평문 |
| **키(Key)** | 암호화/복호화에 사용되는 비밀 정보 |

> 현대 암호 시스템에서 알고리즘 자체는 **공개**되어 있다. 보안은 오직 **키의 비밀성**에 의존한다.

## 대칭키 암호 (Symmetric Key Cryptography)

송신자와 수신자가 **동일한 비밀 키**를 공유한다.

### 고전 암호

| 방식 | 키 공간 | 특징 |
|------|--------|------|
| **Caesar cipher** | 25 (시프트 값 k) | 각 문자를 k만큼 이동. 무차별 대입으로 즉시 해독 |
| **단일치환 암호(Monoalphabetic)** | 26! ≈ 10^26 | 임의 1:1 매핑. 통계 분석으로 해독 가능 (빈도 분석) |
| **다중치환 암호(Polyalphabetic)** | 다중 Caesar 조합 | 같은 문자가 위치에 따라 다르게 암호화. 해독 난이도 향상 |

### 공격 시나리오

- **Ciphertext-only attack**: 암호문만으로 해독 시도 (통계 분석)
- **Known-plaintext attack**: (평문, 암호문) 쌍 일부를 알고 해독
- **Chosen-plaintext attack**: 침입자가 원하는 평문을 암호화시켜 암호문을 얻어 분석

### 블록 암호 (Block Cipher)

메시지를 **k비트 블록** 단위로 나누어 각각 독립적으로 암호화한다.

- k비트 블록의 가능한 매핑 수: **2^k!** (full-table 방식은 비현실적)
- 실제로는 **순열 테이블 + 다중 라운드(n rounds)** 함수를 사용하여 유사 랜덤 순열 시뮬레이션
- 각 라운드에서 입력 비트의 영향이 출력 전체로 확산(diffusion)

| 알고리즘 | 블록 크기 | 키 길이 | 비고 |
|----------|----------|--------|------|
| **DES** | 64비트 | 56비트 | 1초 내 크래킹 가능 (NIST 2001 추정) |
| **3DES** | 64비트 | 168비트 | DES 3회 적용 |
| **AES** | 128비트 | 128/192/256비트 | 128비트 키 해독에 ~149조 년 |

### CBC (Cipher Block Chaining)

동일한 평문 블록이 동일한 암호문을 생성하는 문제를 해결한다:

1. 랜덤 **초기화 벡터(IV)** c(0)를 생성하여 **평문으로** 전송
2. 첫 블록: c(1) = K_S(m(1) ⊕ c(0))
3. i번째 블록: c(i) = K_S(m(i) ⊕ c(i-1))
4. 수신자: m(i) = K_S^-1(c(i)) ⊕ c(i-1)

오버헤드: IV 1개만 추가 전송 → 긴 메시지에서 대역폭 부담 무시 가능

## 공개키 암호 (Public Key Cryptography)

1976년 Diffie-Hellman이 제안. 두 개의 키를 사용한다:
- **공개키 K_B^+**: 모든 사람에게 공개 (암호화용)
- **개인키 K_B^-**: 소유자만 보유 (복호화용)

핵심 성질: K_B^-(K_B^+(m)) = m **그리고** K_B^+(K_B^-(m)) = m (순서 교환 가능)

### RSA 알고리즘

1. 큰 소수 p, q 선택 (각 1024비트 이상 권장)
2. n = pq, z = (p-1)(q-1) 계산
3. e 선택: 1 < e < n, gcd(e, z) = 1
4. d 계산: ed mod z = 1
5. 공개키 (n, e), 개인키 (n, d)

**암호화**: c = m^e mod n  
**복호화**: m = c^d mod n

> 동작 원리: m^(ed) mod n = m^(ed mod z) mod n = m^1 mod n = m (페르마-오일러 정리 활용)

**보안 근거**: n의 소인수분해가 계산적으로 어려움. 양자 컴퓨터가 이를 위협할 수 있으나 (MIT TR 2019), 실용적 실현은 아직 먼 미래.

### RSA의 성능 한계와 세션 키

RSA 지수 연산은 느리므로, 실무에서는 **하이브리드 방식**을 사용:
1. Alice가 랜덤 **세션 키** K_S 생성
2. K_S를 Bob의 공개키로 암호화하여 전송: c = (K_S)^e mod n
3. Bob이 개인키로 K_S 복호화
4. 이후 데이터는 K_S로 대칭키 암호화 (AES 등)

### Diffie-Hellman 키 교환

RSA와 달리 임의 길이 메시지 암호화는 불가하지만, **대칭 세션 키를 안전하게 합의**하는 데 사용된다.
