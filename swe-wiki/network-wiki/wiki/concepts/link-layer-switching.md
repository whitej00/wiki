# 링크 계층 스위칭 (Link-Layer Switching)

**링크 계층 스위치(Link-Layer Switch)**는 수신한 프레임을 MAC 주소 기반으로 적절한 출력 인터페이스로 포워딩하는 **layer-2 패킷 스위치**다. 호스트와 라우터에게 **투명(Transparent)**하게 동작한다.

## 스위치 테이블 (Switch Table)

스위치는 내부에 **스위치 테이블**을 유지하며, 각 엔트리는:

| 필드 | 설명 |
|------|------|
| MAC 주소 | 목적지 어댑터의 MAC 주소 |
| 인터페이스 | 해당 MAC 주소로 향하는 스위치 인터페이스 번호 |
| 시간 | 엔트리가 기록된 시간 |

## 필터링과 포워딩

프레임이 인터페이스 x에 도착하고 목적지 MAC 주소가 DD-DD-DD-DD-DD-DD일 때:

1. **테이블에 엔트리 없음** → 인터페이스 x를 제외한 **모든 인터페이스로 브로드캐스트**
2. **엔트리가 인터페이스 x와 연결** → 목적지가 이미 같은 세그먼트에 있으므로 **필터링(폐기)**
3. **엔트리가 인터페이스 y(≠x)와 연결** → 인터페이스 y의 출력 버퍼로 **포워딩**

## 자가 학습 (Self-Learning)

스위치 테이블은 **자동으로** 구축된다 (plug-and-play):

1. 테이블은 초기에 **비어 있음**
2. 프레임이 인터페이스에 도착할 때마다 **출발지 MAC 주소**, 도착 인터페이스, 현재 시간을 기록
3. **에이징 타임(Aging Time)** 동안 해당 출발지 주소의 프레임이 없으면 엔트리 삭제

이 과정으로 스위치는 네트워크 관리자의 개입 없이 각 MAC 주소의 위치를 학습한다.

## 스위치의 장점

| 장점 | 설명 |
|------|------|
| **충돌 제거** | 프레임을 버퍼링하고 한 번에 하나씩 포워딩 → 충돌 없음 |
| **이종 링크 지원** | 각 링크가 다른 속도/매체로 동작 가능 (예: 1 Gbps + 100 Mbps 혼재) |
| **관리 용이** | 오작동 어댑터(jabbering adapter) 자동 감지/차단, 대역폭/충돌률 통계 수집 |
| **전이중(Full-Duplex)** | 모든 인터페이스에서 동시 송수신 가능 |

## 스위치 vs 라우터

| 특성 | 허브 | 스위치 | 라우터 |
|------|------|--------|--------|
| 트래픽 격리 | No | Yes | Yes |
| Plug-and-Play | Yes | Yes | No |
| 최적 경로 라우팅 | No | No | Yes |
| 동작 계층 | 물리(L1) | 링크(L2) | 네트워크(L3) |

**스위치의 단점**:
- 브로드캐스트 프레임의 순환 방지를 위해 **스패닝 트리**로 활성 토폴로지 제한
- 대규모 스위치 네트워크에서 ARP 트래픽 폭증 가능
- **브로드캐스트 스톰** 취약 — 한 호스트가 무한 브로드캐스트 시 전체 네트워크 마비

**라우터의 단점**:
- Plug-and-play가 아님 (IP 주소 설정 필요)
- L3까지 처리하므로 패킷 당 처리 시간이 더 김

> 소규모 네트워크(수백 호스트)에서는 스위치, 대규모 네트워크(수천 호스트 이상)에서는 스위치 + 라우터 조합이 일반적이다.

## 보안: 스위치 포이즈닝 (Switch Poisoning)

공격자가 위조된 출발지 MAC 주소의 프레임을 대량으로 전송하여 스위치 테이블을 가짜 엔트리로 채움 → 정상 MAC 주소의 엔트리가 밀려남 → 스위치가 대부분의 프레임을 브로드캐스트 → 공격자가 스니핑 가능.
