# 캡슐화 (Encapsulation)

각 프로토콜 계층이 상위 계층으로부터 받은 데이터에 **자신의 헤더 정보를 추가**하여 하위 계층으로 전달하는 과정.

## 동작 원리

송신 호스트에서 데이터가 프로토콜 스택을 내려가며 각 계층에서 헤더가 추가된다:

```
Application:  [         M         ]        ← 메시지
Transport:    [  H_t  |    M      ]        ← 세그먼트
Network:      [ H_n | H_t |  M   ]        ← 데이터그램
Link:         [H_l| H_n | H_t | M]        ← 프레임
```

- **Application 계층**: 메시지(M) 생성
- **Transport 계층**: 메시지에 전송 계층 헤더(H_t) 추가 → **세그먼트(segment)**
  - 수신 측에서 적절한 애플리케이션으로 전달하기 위한 정보, 오류 검출 비트 포함
- **Network 계층**: 세그먼트에 네트워크 계층 헤더(H_n) 추가 → **데이터그램(datagram)**
  - 출발지/목적지 종단 시스템 주소 포함
- **Link 계층**: 데이터그램에 링크 계층 헤더(H_l) 추가 → **프레임(frame)**

수신 측에서는 역순으로 **역캡슐화(de-encapsulation)**가 진행된다.

## 각 장비에서의 처리

- **링크 계층 스위치**: 프레임을 역캡슐화 → 링크/물리 계층만 처리 → 재캡슐화
- **라우터**: 데이터그램까지 역캡슐화 → 네트워크 계층까지 처리 → 재캡슐화
- **호스트**: 모든 5개 계층 처리

## 핵심 용어

- **헤더 필드(Header field)**: 해당 계층의 프로토콜 정보
- **페이로드 필드(Payload field)**: 상위 계층으로부터 받은 패킷 (즉, 상위 계층의 헤더 + 데이터)
