# WiFi: IEEE 802.11 무선 LAN

IEEE 802.11 무선 LAN, 통칭 **WiFi**는 오늘날 가장 널리 사용되는 무선 접속 기술이다. 가정, 직장, 카페, 공항 등에서 인터넷 접속의 주요 수단으로 사용된다.

## 한 줄 요약

CSMA/CA 기반 매체 접근, link-layer ACK, 4개 주소 필드를 갖는 프레임을 사용하여 무선 호스트와 AP 간 통신을 제공하는 링크 계층 프로토콜.

## 802.11 표준 변천

| 표준 | 연도 | 최대 속도 | 범위 | 주파수 |
|------|------|---------|------|--------|
| 802.11b | 1999 | 11 Mbps | 30 m | 2.4 GHz |
| 802.11g | 2003 | 54 Mbps | 30 m | 2.4 GHz |
| 802.11n (WiFi 4) | 2009 | 600 Mbps | 70 m | 2.4/5 GHz |
| 802.11ac (WiFi 5) | 2013 | 3.47 Gbps | 70 m | 5 GHz |
| 802.11ax (WiFi 6) | 2020 | 14 Gbps | 70 m | 2.4/5 GHz |
| 802.11af | 2014 | 35-560 Mbps | 1 km | 54-790 MHz |
| 802.11ah | 2017 | 347 Mbps | 1 km | 900 MHz |

802.11n부터 **MIMO**(Multiple Input Multiple Output) 안테나 기술을 채택하며, 802.11ac/ax는 **beam forming**을 지원한다. 모든 표준은 동일한 프레임 포맷과 CSMA/CA MAC 프로토콜을 공유하며 하위 호환된다.

## 아키텍처

### BSS (Basic Service Set)

802.11의 기본 구성 단위. 하나의 **AP(Access Point, 기지국)**와 하나 이상의 무선 스테이션으로 구성된다.

- **Infrastructure wireless LAN**: AP가 유선 네트워크(스위치/라우터)에 연결되어 인터넷 접속 제공
- **Ad hoc network**: AP 없이 무선 장치끼리 직접 통신하는 네트워크

### 채널과 SSID

- AP에는 **SSID**(Service Set Identifier)와 **채널 번호**가 할당된다
- 2.4 GHz 대역: 85 MHz 안에 11개 부분 겹침 채널, **비중첩 채널은 1, 6, 11 세 개**만 존재
- 5 GHz 대역: 더 많은 비중첩 채널 제공, 간섭이 적지만 도달 거리 짧음

### Association (연관)

무선 장치가 인터넷을 사용하려면 정확히 하나의 AP와 **연관(associate)**해야 한다.

**Passive scanning**: AP가 주기적으로 보내는 **beacon frame**(SSID, MAC 주소 포함)을 스캔하여 AP를 발견한다.

**Active scanning**: 무선 장치가 **probe request** 프레임을 브로드캐스트하고, 범위 내 AP들이 probe response로 응답한다.

AP 선택 후 **association request/response** 교환으로 연관이 수립되며, 이후 DHCP를 통해 IP 주소를 획득한다.

인증 방식: MAC 주소 기반, 사용자명/비밀번호(WPA), 인증 서버(RADIUS/DIAMETER) 연동.

## MAC 프로토콜: CSMA/CA

802.11은 **CSMA with Collision Avoidance (CSMA/CA)** 방식을 사용한다. 유선 이더넷의 CSMA/CD와 두 가지 핵심 차이가 있다:

1. **충돌 회피(Collision Avoidance)** 사용 — 충돌 검출(Detection)이 불가능하기 때문
2. **Link-layer ACK** 사용 — 높은 BER로 인해 프레임 도착을 확인해야 함

### CSMA/CA 절차

1. 채널이 유휴하면 **DIFS**(Distributed Inter-Frame Space) 대기 후 프레임 전송
2. 채널이 사용 중이면 **binary exponential backoff** 값을 선택하고, 채널 유휴 시에만 카운트다운 (사용 중이면 동결)
3. 카운터가 0이 되면 (채널 유휴 시에만 발생) 프레임 전체를 전송하고 ACK 대기
4. ACK 수신 시 성공; 미수신 시 더 큰 백오프 범위에서 재시도

**핵심**: CSMA/CD에서는 채널이 유휴하면 즉시 전송하지만, CSMA/CA에서는 유휴 상태에서도 백오프 중이면 카운트다운을 계속한다. 이는 **802.11에서 충돌 시 프레임 전체가 낭비**되기 때문에 충돌을 최대한 피하기 위함이다.

### Link-layer ACK

수신 스테이션이 CRC를 통과한 프레임을 받으면 **SIFS**(Short Inter-Frame Spacing) 후 ACK를 반환한다. 일정 횟수 재전송 후에도 ACK가 없으면 포기한다.

### RTS/CTS (Request to Send / Clear to Send)

**Hidden terminal 문제**를 완화하기 위한 선택적 예약 메커니즘:

1. 송신자가 **RTS** 프레임을 AP에 전송 (전송할 DATA + ACK에 필요한 시간 명시)
2. AP가 **CTS** 프레임을 브로드캐스트 — 송신자에게 전송 허가, 다른 스테이션에게 해당 시간 동안 전송 금지 통보
3. RTS/CTS 프레임은 짧으므로 충돌 시 낭비가 적다

실제로는 대부분의 무선 스테이션이 RTS threshold를 프레임 최대 길이 이상으로 설정하여, RTS/CTS를 생략하는 경우가 많다.

## 802.11 프레임 구조

```
| Frame Control (2) | Duration (2) | Address 1 (6) | Address 2 (6) | Address 3 (6) | Seq Control (2) | Address 4 (6) | Payload (0-2312) | CRC (4) |
```

### 4개 주소 필드

이더넷 프레임(2개 주소)과 달리 802.11은 **4개 주소 필드**를 갖는다:

| 주소 | 역할 (Infrastructure mode) |
|------|---------------------------|
| Address 1 | 수신 무선 인터페이스의 MAC 주소 |
| Address 2 | 송신 무선 인터페이스의 MAC 주소 |
| Address 3 | BSS가 연결된 라우터 인터페이스의 MAC 주소 |
| Address 4 | Ad hoc 모드에서 AP 간 포워딩 시 사용 |

**Address 3의 역할**: AP는 링크 계층 장치로 IP를 이해하지 못한다. R1 → H1으로 데이터그램을 보낼 때:
- R1이 이더넷 프레임(src: R1 MAC, dst: H1 MAC)을 전송
- AP가 이를 802.11 프레임으로 변환: Address 1 = H1 MAC, Address 2 = AP MAC, **Address 3 = R1 MAC**
- H1은 Address 3을 통해 원래 송신 라우터 인터페이스를 식별

### 기타 필드

- **Sequence number**: ACK 손실 시 중복 프레임 구별 (전송 계층 TCP와 동일한 역할을 링크 계층에서 수행)
- **Duration**: RTS/CTS와 데이터 프레임에서 채널 예약 시간 명시
- **Frame control**: type/subtype(RTS, CTS, ACK, data), to/from AP, WEP 비트 등

## Same-Subnet Mobility

같은 IP 서브넷 내의 BSS 간 이동 시:

1. H1이 AP1에서 멀어지면서 AP2의 beacon 수신 → 동일 SSID면 AP1과 연관 해제, AP2와 연관
2. **IP 주소와 TCP 연결 유지** (같은 서브넷이므로)
3. 스위치의 포워딩 테이블 갱신: AP2가 H1의 MAC 주소로 broadcast 이더넷 프레임 전송 → 스위치가 self-learning으로 테이블 갱신

서로 다른 서브넷 간 이동 시에는 [이동성 관리](../concepts/mobility-management.md) 프로토콜이 필요하다.

## 고급 기능

### Rate Adaptation

SNR 변화에 따라 변조 기법을 동적으로 선택한다:
- 연속 2회 ACK 미수신 → 전송률 한 단계 하향
- 연속 10회 ACK 수신 또는 타이머 만료 → 전송률 한 단계 상향
- TCP의 혼잡 제어와 동일한 **"probing"** 철학

### Power Management

- 스테이션이 power management 비트를 설정하고 sleep 진입
- AP가 해당 스테이션 행 프레임을 버퍼링
- 스테이션은 AP의 **beacon frame** (약 100 ms 간격) 직전에 깨어나 (250 μs) 버퍼 목록 확인
- 버퍼된 프레임이 없으면 다시 sleep → **99% 시간 절전** 가능
